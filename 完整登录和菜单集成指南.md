# 完整登录和菜单集成指南

## ✅ 所有修复已完成

### 已实现的 API 接口

#### 1. 登录接口
```
POST /api/login/
请求体: { "userName": "Admin", "password": "123456" }
响应: {
  "code": 200,
  "msg": "登录成功",
  "data": {
    "token": "eyJ...",           # Access token (1小时有效期)
    "refreshToken": "eyJ..."    # Refresh token (7天有效期)
  }
}
```

#### 2. 获取用户信息接口
```
GET /api/user/info/
请求头: Authorization: Bearer {token}
响应: {
  "code": 200,
  "msg": "获取成功",
  "data": {
    "userId": 1,
    "userName": "Admin",
    "email": "admin@example.com",
    "avatar": "",
    "roles": ["R_ADMIN"],        # ✨ 返回用户角色
    "buttons": []
  }
}
```

#### 3. 获取菜单列表接口（新增）
```
GET /api/v3/system/menus/
请求头: Authorization: Bearer {token}
响应: {
  "code": 200,
  "msg": "获取成功",
  "data": [                      # ✨ 菜单数组
    {
      "path": "/dashboard",
      "name": "Dashboard",
      "component": "Dashboard",
      "meta": {
        "title": "仪表盘",
        "icon": "dashboard",
        "roles": ["R_ADMIN", "R_USER"]
      }
    },
    {
      "path": "/system",
      "name": "System",
      "component": "Layout",
      "meta": {
        "title": "系统管理",
        "icon": "setting",
        "roles": ["R_ADMIN"]
      },
      "children": [...]
    }
  ]
}
```

## 🐛 常见错误和解决方案

### 错误 1：404 Not Found on /api/login/
**原因**：Django 服务器没有启动或路由配置不对
**解决**：
```bash
python manage.py runserver 0.0.0.0:8000
```

### 错误 2：401 Unauthorized on /api/user/info/
**原因**：
- Token 格式错误（应该是 `Bearer {token}` 格式）
- Token 已过期
- Django 没有正确配置 JWT

**解决**：
- 硬刷新浏览器 (Cmd+Shift+R)
- 重新登录获取新的 token
- 检查后端 JWT 配置

### 错误 3：[RouteGuard] 用户无权限访问路径
**原因**：菜单列表为空或没有包含目标路由
**解决**：
- 确保 `/api/v3/system/menus/` 返回了有效的菜单数据
- 检查用户角色是否包含在菜单的 `meta.roles` 中
- 查看浏览器开发者工具中的 Network 标签，检查菜单返回值

### 错误 4：Missing required prop "index" (ElMenuItem)
**原因**：动态路由配置不完整
**解决**：
- 确保菜单数据结构完整
- 检查前端是否正确接收到菜单列表
