## 🔑 关键文件详解

### 1️⃣ `deploy.sh` - 自动化部署脚本（最重要！）

**功能：**
- 检查系统环境（Python、Node.js、Nginx）
- 创建虚拟环境并安装 Django 依赖
- 执行数据库迁移
- 建超级用户（交互式）
- 收集静态文件
- 构建 Vue 前端
- 生成 Gunicorn 配置

**使用：**
```bash
chmod +x deploy.sh
sudo bash deploy.sh
```

**输出：**
- ✓ 虚拟环境：`venv/`
- ✓ 数据库表：正式环境数据库
- ✓ 静态文件：`static/` 目录
- ✓ 前端构建：`art-design-pro/dist/`

---

### 2️⃣ `seaboat-gunicorn.service` - 系统服务管理

**功能：**
- 由 systemd 管理 Gunicorn 进程
- 自动启动和重启应用
- 集中管理日志和环境变量

**安装步骤：**
```bash
sudo cp seaboat-gunicorn.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable seaboat-gunicorn
sudo systemctl start seaboat-gunicorn
```

**常用命令：**
```bash
sudo systemctl status seaboat-gunicorn    # 查看状态
sudo systemctl restart seaboat-gunicorn   # 重启服务
sudo journalctl -u seaboat-gunicorn -f    # 查看日志
```

---

### 3️⃣ `nginx.conf` - Web 服务器配置

**功能：**
- 前端路由（Vue SPA）→ `/` (index.html)
- API 反向代理 → `/api/` (Gunicorn socket)
- HTTPS 重定向
- 静态文件缓存
- 请求日志

**部署：**
```bash
# 方式A：宝塔面板 UI
# 网站 → yunpingtai.cc → 配置文件 → 粘贴内容

# 方式B：命令行
sudo cp nginx.conf /usr/local/nginx/conf/vhost/yunpingtai.cc.conf
sudo systemctl reload nginx
```

---

### 4️⃣ `settings_prod.py` - Django 生产配置

**包含：**
- DEBUG = False（关闭调试）
- MySQL/PostgreSQL 连接配置
- 静态文件路径
- 日志配置（轮转）
- CORS 和 CSRF 安全设置
- JWT token 配置
- 文件上传限制（500MB）

**注意：** 依赖 `.env` 文件中的环境变量：
- `SECRET_KEY`
- `DB_PASSWORD`
- `DEBUG`（可选）

---

### 5️⃣ `.env.example` - 环境变量模板

**使用步骤：**
```bash
cp .env.example .env
nano .env  # 编辑

# 必改项：
# SECRET_KEY=生成的密钥（50字符）
# DB_PASSWORD=数据库密码
```

**主要变量：**
- `SECRET_KEY` - Django 加密密钥
- `DB_PASSWORD` - 数据库密码
- `DB_NAME`, `DB_USER`, `DB_HOST`, `DB_PORT` - 数据库连接
- `GUNICORN_WORKERS` - Gunicorn 工作进程数（推荐4）
- `LANGUAGE_CODE`, `TIME_ZONE` - 中文和时区配置

## 📞 问题排查流程

### 无法访问网站？
```bash
# 1. 检查域名 DNS
nslookup yunpingtai.cc

# 2. 检查 Nginx 状态
sudo systemctl status nginx
tail -f /home/wwwroot/yunpingtai.cc/logs/nginx_error.log

# 3. 检查 Gunicorn 状态
sudo systemctl status seaboat-gunicorn
sudo journalctl -u seaboat-gunicorn -n 50
```

### API 返回 502？
```bash
# 检查 Gunicorn 是否运行
sudo systemctl status seaboat-gunicorn

# 重启
sudo systemctl restart seaboat-gunicorn

# 查看错误日志
tail -f /home/wwwroot/yunpingtai.cc/logs/gunicorn_error.log
```

### 数据库连接失败？
```bash
# 测试数据库连接
mysql -u seaboat_user -p -h localhost seaboat_prod

# 检查 .env 文件
cat /home/wwwroot/yunpingtai.cc/.env | grep DB
```
