# 🌊 无人船监测系统 - 项目使用说明

## 📋 项目结构

```
seabot/
├── art-design-pro/          # 前端 Vue 3 + TypeScript 项目
│   ├── src/
│   │   ├── api/             # API 接口定义
│   │   │   ├── auth.ts      # 用户认证接口
│   │   │   ├── dashboard.ts # 仪表板数据接口
│   │   │   └── system-manage.ts # 系统管理接口
│   │   ├── views/           # 页面组件
│   │   │   ├── dashboard/   # 仪表板（工作台、分析）
│   │   │   ├── auth/        # 登录注册页面
│   │   │   └── system/      # 系统管理页面
│   │   ├── store/           # Pinia 状态管理
│   │   └── utils/
│   │       └── http/        # HTTP 请求工具
│   └── package.json
│
├── back-end/                # Django 后端项目
│   ├── seadrone/            # Django 项目配置
│   │   ├── settings.py      # 项目设置
│   │   ├── urls.py          # URL 路由
│   │   └── wsgi.py
│   ├── accounts/            # 用户认证和仪表板应用
│   │   ├── models.py        # 用户、仪表板、设备模型
│   │   ├── views.py         # API 视图
│   │   └── urls.py          # 路由
│   ├── monitoring/          # 数据监测应用（水质、营养盐）
│   │   ├── models.py        # WaterQualityData, NutrientData
│   │   ├── views.py         # 获取/上传数据接口
│   │   └── urls.py          # 路由
│   ├── manage.py            # Django 管理脚本
│   ├── db.sqlite3           # SQLite 数据库
│   ├── myenv/               # Python 虚拟环境
│   ├── create_monitoring_data.py    # 创建示例监测数据
│   ├── create_dashboard_data.py     # 创建仪表板数据

```

## 🚀 快速启动指南

### 1️⃣ 启动后端服务（Python/Django）

```bash
# 进入后端目录
cd seabot/back-end/seadrone

# 激活虚拟环境
source ../../.venv/bin/activate

# 启动 Django 开发服务器（端口 8000）
python3 manage.py runserver 0.0.0.0:8000
```

**服务启动后，访问以下地址：**
- 🔹 API 测试：http://localhost:8000/api/
- 🔹 Django 后台管理：http://localhost:8000/admin/

---

### 2️⃣ 启动前端服务（Vue 3）

**在新的终端窗口中：**

```bash
# 进入前端目录
cd seabot/art-design-pro

# 激活虚拟环境（可选）
source ../.venv/bin/activate

# 安装依赖（首次运行）
pnpm install

# 启动 Vite 开发服务器（端口 5173）
pnpm dev
```

**前端访问地址：**
- 🔹 应用首页：http://localhost:5173

## 🛠️ Python 后端管理命令

### 数据库相关

```bash
# 进入后端目录
cd seabot/back-end/seadrone
source ../../.venv/bin/activate

# 查看数据库迁移状态
python3 manage.py showmigrations

# 创建新的迁移文件（修改模型后）
python3 manage.py makemigrations

# 应用迁移到数据库
python3 manage.py migrate

# 重置数据库
python3 manage.py flush
```

### 用户管理

```bash
# 创建超级用户（管理员）
python3 manage.py createsuperuser

# 修改用户密码
python3 manage.py changepassword username
```

### 创建示例数据

```bash
# 创建仪表板数据（销售、用户增长）
python3 create_dashboard_data.py

# 创建示例设备（DL-3026、DL-3022）
python3 create_sample_devices.py

# 创建监测数据（水质、营养盐）
python3 create_monitoring_data.py
```

### Django 后台管理

访问：http://localhost:8000/admin/

**默认用户信息（如已创建）：**
- 用户名：admin
- 密码：需要通过 `createsuperuser` 创建

## 🐛 常见问题排查

### 1. 端口已被占用

```bash
# 查看占用 8000 端口的进程
lsof -i :8000

# 杀死进程
kill -9 <PID>

# 或使用不同端口启动
python3 manage.py runserver 8001
```

### 2. 数据库错误

```bash
# 删除数据库重新初始化
rm db.sqlite3
python3 manage.py migrate
python3 create_dashboard_data.py
python3 create_monitoring_data.py
```

### 3. 权限问题（401/403）

- 确保 API 请求包含 `Authorization: Bearer {token}` 头
- 查看 monitoring 应用的数据接口是否移除了认证要求

### 4. CORS 错误

- 检查 `settings.py` 中的 `CORS_ALLOWED_ORIGINS` 配置
- 确保包含 `http://localhost:5173`

## 🎯 开发工作流

### 1. 修改后端代码

1. 编辑 `back-end/seadrone/` 中的 Python 文件
2. Django 开发服务器会自动重新加载
3. 使用 curl 或前端测试 API

### 2. 修改前端代码

1. 编辑 `art-design-pro/src/` 中的 Vue/TypeScript 文件
2. Vite 开发服务器会自动热更新
3. 刷新浏览器查看效果

### 3. 添加新的 API 端点

1. 在 `back-end/seadrone/` 中创建或修改模型（models.py）
2. 创建视图函数（views.py）
3. 添加路由（urls.py）
4. 运行迁移：`python3 manage.py makemigrations && python3 manage.py migrate`
5. 在前端 `art-design-pro/src/api/` 中创建对应的 API 函数
