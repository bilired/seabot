## 🚀 部署步骤概览（5个阶段）

### 第一步：准备工作（宝塔面板）⏱️ 15分钟

1. **安装 Python 3.11**（如上面的解决方案）
2. **添加网站**
   - 宝塔 → 网站 → 添加网站
   - 域名：`yunpingtai.cc` 和 `www.yunpingtai.cc`
3. **申请 SSL 证书**
   - 点击网站 → SSL → Let's Encrypt 免费证书
   - 勾选"自动续期"
4. **创建数据库**
   - 宝塔 → 数据库 → 添加数据库
   - 数据库名：`seaboat_prod`
   - 用户名：`seaboat_user`
   - 设置强密码并保存

### 第二步：上传和配置（SSH 或宝塔文件管理）⏱️ 5分钟

```bash
cd /home/wwwroot/yunpingtai.cc

# 上传代码（选一种）
# 选项A：Git 克隆
git clone <你的仓库> .

# 选项B：宝塔面板上传压缩包并解压

# 配置环境变量
cp .env.example .env
nano .env
# 编辑以下值：
# SECRET_KEY = （运行命令生成：python3 -c "import secrets; print(secrets.token_urlsafe(50))"）
# DB_PASSWORD = （你在宝塔创建数据库时设置的密码）
```

### 第三步：自动化部署（SSH）⏱️ 20-30分钟

```bash
cd /home/wwwroot/yunpingtai.cc
chmod +x deploy.sh
sudo bash deploy.sh

# 脚本会自动：
# ✓ 检查 Python/Node.js 环境
# ✓ 创建虚拟环境
# ✓ 安装依赖
# ✓ 执行数据库迁移
# ✓ 创建超级用户（交互输入）
# ✓ 收集静态文件
# ✓ 构建前端
```

### 第四步：启动服务（SSH）⏱️ 3分钟

```bash
# 安装 Gunicorn 系统服务
sudo cp seaboat-gunicorn.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable seaboat-gunicorn
sudo systemctl start seaboat-gunicorn
sudo systemctl status seaboat-gunicorn  # 验证运行

# 配置 Nginx（两种方式）
# 方式A：宝塔面板 UI
#   → 网站 → yunpingtai.cc → 配置文件 → 粘贴 nginx.conf 内容 → 保存

# 方式B：命令行
sudo cp nginx.conf /usr/local/nginx/conf/vhost/yunpingtai.cc.conf
sudo systemctl reload nginx
```

### 第五步：验证部署（浏览器）⏱️ 2分钟

```bash
# 后端 API（需要 token）
curl https://yunpingtai.cc/api/drone/list/ \
  -H "Authorization: Bearer <your_token>" -k

# 前端访问
打开浏览器访问：https://yunpingtai.cc
→ 应看到登录页面
→ 用超级用户账号登录
→ 导航到"无人船设备管理"页面验证

# 查看应用日志
tail -f /home/wwwroot/yunpingtai.cc/logs/django.log
```

## 📞 快速问题排查

### 问题：502 Bad Gateway
```bash
sudo systemctl status seaboat-gunicorn  # 检查是否运行
sudo systemctl restart seaboat-gunicorn  # 重启
tail -f /home/wwwroot/yunpingtai.cc/logs/gunicorn_error.log  # 查看错误
```

### 问题：数据库连接失败
```bash
# 验证连接信息
mysql -u seaboat_user -p -h localhost seaboat_prod
# 检查 .env 中的密码
cat /home/wwwroot/yunpingtai.cc/.env | grep DB_PASSWORD
```

### 问题：静态文件 404
```bash
# 重新收集静态文件
cd /home/wwwroot/yunpingtai.cc/back-end/seadrone
python manage.py collectstatic --noinput --settings=seadrone.settings_prod
```

### 问题：前端显示异常
```bash
# 检查是否构建完成
ls /home/wwwroot/yunpingtai.cc/art-design-pro/dist/index.html

# 如果不存在，重新构建
cd /home/wwwroot/yunpingtai.cc/art-design-pro
npm run build
```

更详细的问题排查请查看 **部署检查清单.md** 中的 FAQ 部分。
