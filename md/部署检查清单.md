# 宝塔面板部署检查清单 - yunpingtai.cc

## 📝 部署前准备清单

### 域名和 SSL
- [ ] 域名 `yunpingtai.cc` 已指向服务器 IP
- [ ] 宝塔面板中已添加网站 `yunpingtai.cc`
- [ ] Let's Encrypt SSL 证书已申请
- [ ] SSL 自动续期已启用

### 服务器环境
- [ ] CentOS 8.2.2004 系统
- [ ] Python 3.11+ 已安装（需通过宝塔软件商店安装）
- [ ] Node.js 18+ 已安装（需通过宝塔软件商店安装）
- [ ] Nginx 已安装并正常运行
- [ ] MySQL 8.0 或 PostgreSQL 已安装

### 数据库准备
- [ ] 在宝塔面板创建数据库：`seaboat_prod`
- [ ] 创建数据库用户：`seaboat_user`
- [ ] 设置强密码并保存
- [ ] 记下数据库密码（将在部署时使用）

### 代码准备
- [ ] 将代码克隆或上传到 `/home/wwwroot/yunpingtai.cc/`
- [ ] 项目结构包含：
  - `back-end/seadrone/` - Django 项目
  - `art-design-pro/` - Vue 前端
  - `deploy.sh` - 部署脚本

## 🚀 部署步骤

### 步骤 1：生成 SECRET_KEY

```bash
ssh root@yunpingtai.cc

# 登录服务器后，生成 Django SECRET_KEY
python3 -c "import secrets; print(secrets.token_urlsafe(50))"

# 输出示例：
# K-8_xA9Z5vL2mN3bQ7wE1rT4uY6hJ8kL9pO0qRsT5uV6w
```

**保存这个值！** 稍后会用到。

---

### 步骤 2：设置环境变量

```bash
cd /home/wwwroot/yunpingtai.cc

# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件
nano .env

# 修改以下值：
# SECRET_KEY=你刚才生成的值
# DB_PASSWORD=你在宝塔创建的数据库密码
```

**保存文件：** `Ctrl+X` → `Y` → `Enter`

---

### 步骤 3：运行自动化部署脚本

```bash
# 确保在项目根目录
cd /home/wwwroot/yunpingtai.cc

# 赋予执行权限
chmod +x deploy.sh

# 运行部署脚本（需要 sudo 权限）
sudo bash deploy.sh
```

脚本会自动：
- ✅ 检查 Python 和 Node.js 环境
- ✅ 创建虚拟环境并安装依赖
- ✅ 执行数据库迁移
- ✅ 创建超级用户
- ✅ 收集静态文件
- ✅ 构建前端
- ✅ 生成 Gunicorn 配置

---

### 步骤 4：安装 Gunicorn 系统服务

```bash
# 复制 systemd 服务文件
sudo cp seaboat-gunicorn.service /etc/systemd/system/

# 重新加载 systemd
sudo systemctl daemon-reload

# 启用开机自启
sudo systemctl enable seaboat-gunicorn

# 启动服务
sudo systemctl start seaboat-gunicorn

# 检查状态
sudo systemctl status seaboat-gunicorn
```

---

### 步骤 5：配置 Nginx（使用宝塔面板）

1. 登录宝塔面板（`http://your-ip:8888`）
2. 左侧菜单 → **网站** → 找到 `yunpingtai.cc` → **配置文件**
3. 将 [nginx.conf](nginx.conf) 的content部分复制到宝塔编辑框中
4. 保存并重启 Nginx

**或者命令行方式：**
```bash
sudo cp nginx.conf /usr/local/nginx/conf/vhost/yunpingtai.cc.conf
sudo systemctl reload nginx
```

## 🔧 常见问题

### Q1: 502 Bad Gateway 错误

**原因：** Gunicorn 未运行或 socket 权限问题

**解决方案：**
```bash
# 检查 Gunicorn 状态
sudo systemctl status seaboat-gunicorn

# 查看 Gunicorn 错误日志
sudo journalctl -u seaboat-gunicorn -n 50

# 重启 Gunicorn
sudo systemctl restart seaboat-gunicorn

# 检查 socket 文件权限
ls -la /home/wwwroot/yunpingtai.cc/gunicorn.sock

# 应该是 www:www666 权限
```

---

### Q2: 数据库连接错误

**原因：** 数据库密码错误或 MySQL/PostgreSQL 未运行

**解决方案：**
```bash
# 检查 MySQL 状态
sudo systemctl status mysqld

# 测试数据库连接
mysql -u seaboat_user -p -h localhost seaboat_prod

# 查看 Django 日志
tail -f /home/wwwroot/yunpingtai.cc/logs/django.log

# 确认 .env 文件中的密码正确
cat /home/wwwroot/yunpingtai.cc/.env | grep DB_PASSWORD
```

---

### Q3: CSRF 或 CORS 错误

**原因：** 前端域名未在后端允许列表中

**解决方案：**
1. 编辑 `/home/wwwroot/yunpingtai.cc/back-end/seadrone/seadrone/settings_prod.py`
2. 查找 `CORS_ALLOWED_ORIGINS` 和 `CSRF_TRUSTED_ORIGINS`
3. 确保包含 `https://yunpingtai.cc` 和 `https://www.yunpingtai.cc`
4. 重启 Gunicorn：`sudo systemctl restart seaboat-gunicorn`

---

### Q4: 静态文件 404

**原因：** 未收集静态文件或路径配置错误

**解决方案：**
```bash
cd /home/wwwroot/yunpingtai.cc/back-end/seadrone

# 重新收集静态文件
python manage.py collectstatic --noinput --settings=seadrone.settings_prod --clear

# 检查静态文件目录
ls -la /home/wwwroot/yunpingtai.cc/static/

# 应有 Django admin、REST Framework 等文件夹
```

---

### Q5: 上传视频文件时 413 错误

**原因：** Nginx client_max_body_size 过小

**解决方案：**
在 Nginx 配置中添加（已包含在 nginx.conf 中）：
```nginx
client_max_body_size 500M;
```

---

### Q6: HTTPS 证书错误

**原因：** Let's Encrypt 证书过期或未正确配置

**解决方案：**
```bash
# 在宝塔面板手动续期
# 或使用 certbot：
sudo certbot renew --dry-run  # 测试

sudo certbot renew             # 实际更新

# 重启 Nginx
sudo systemctl reload nginx
```

## 🔐 安全加固

1. **SSH 密钥认证**
   ```bash
   ssh-keygen -t ed25519
   ssh-copy-id -i ~/.ssh/id_ed25519.pub root@yunpingtai.cc
   ```

2. **禁用 root 登录**
   ```bash
   sudo sed -i 's/#PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config
   sudo systemctl reload sshd
   ```

3. **防火墙配置**
   ```bash
   sudo firewall-cmd --permanent --add-service=http
   sudo firewall-cmd --permanent --add-service=https
   sudo firewall-cmd --permanent --add-port=8888/tcp  # 宝塔面板
   sudo firewall-cmd --reload
   ```

4. **更新系统**
   ```bash
   sudo yum update -y
   sudo yum upgrade -y
   ```
