## 服务器端部署（25分钟）

### 步骤 1：系统准备（在宝塔面板完成）
```
✓ 添加网站：yunpingtai.cc
✓ 申请 SSL 证书（Let's Encrypt）
✓ 创建数据库：seaboat_prod
✓ 创建用户：seaboat_user + 强密码
✓ 从软件商店安装：Python 3.11, Node.js 18+
```

### 步骤 2：SSH 登录
```bash
ssh root@yunpingtai.cc  # 或使用宝塔面板的终端

cd /home/wwwroot/yunpingtai.cc
```

### 步骤 3：生成密钥（1分钟）
```bash
python3 -c "import secrets; print(secrets.token_urlsafe(50))"

# 复制输出，保存备用
# KcVfj85wGWGvGbde7PdqCHbHZThgoVaNSQlmk3zLzLCBdEhg2gfOXSmzFhY-bEoIsUM
```

### 步骤 4：配置环境变量（2分钟）
```bash
cp .env.example .env
nano .env

# 修改以下 3 行：
# SECRET_KEY=粘贴上面生成的密钥
# DB_PASSWORD=你的数据库密码（在宝塔创建时设置的）
# DB_NAME=seaboat_prod

# 保存：Ctrl+X → Y → Enter
```

### 步骤 5：运行部署脚本（15-20分钟）
```bash
chmod +x deploy.sh
sudo bash deploy.sh

# 脚本会自动：
# ✓ 安装 Python 依赖
# ✓ 执行数据库迁移
# ✓ 创建超级用户（按提示输入）
# ✓ 构建前端
# ✓ 生成配置文件
```

### 步骤 6：启动 Gunicorn 服务（2分钟）
```bash
sudo cp seaboat-gunicorn.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable seaboat-gunicorn
sudo systemctl start seaboat-gunicorn

# 检查状态
sudo systemctl status seaboat-gunicorn

# 应显示 "Active (running)"
```

### 步骤 7：配置 Nginx（1分钟）
**方式 A：宝塔面板 GUI**
1. 打开宝塔面板 → 网站 → yunpingtai.cc → 配置文件
2. 复制 [nginx.conf](nginx.conf) 内容到编辑框
3. 保存并重启 Nginx

**方式 B：命令行**
```bash
sudo cp nginx.conf /usr/local/nginx/conf/vhost/yunpingtai.cc.conf
sudo systemctl reload nginx
```

### 步骤 8：验证部署（2分钟）
```bash
# 后端 API
curl -X GET https://yunpingtai.cc/api/drone/list/ \
  -H "Authorization: Bearer <token>" -k

# 前端
打开浏览器：https://yunpingtai.cc
→ 应看到登录页面

# 日志检查
tail -f /home/wwwroot/yunpingtai.cc/logs/django.log
```

## 🔧 常用运维命令

### 查看服务状态
```bash
# Gunicorn
sudo systemctl status seaboat-gunicorn

# Nginx
sudo systemctl status nginx

# MySQL
sudo systemctl status mysqld
```

### 查看日志
```bash
# 实时日志
tail -f /home/wwwroot/yunpingtai.cc/logs/django.log

# Gunicorn 错误
sudo journalctl -u seaboat-gunicorn -f

# Nginx 错误
tail -f /home/wwwroot/yunpingtai.cc/logs/nginx_error.log
```

### 重启服务
```bash
sudo systemctl restart seaboat-gunicorn  # 重启后端
sudo systemctl reload nginx              # 重新加载前端配置
```

### 更新前端
```bash
cd /home/wwwroot/yunpingtai.cc/art-design-pro
git pull origin main  # 或从本地上传新代码
npm run build         # 重新构建
# Nginx 会自动服务新的 dist 文件
```

### 更新后端
```bash
cd /home/wwwroot/yunpingtai.cc/back-end/seadrone
git pull origin main  # 或从本地上传新代码
source ../../venv/bin/activate
pip install -r requirements.txt  # 安装新依赖（如有）
python manage.py migrate --settings=seadrone.settings_prod
sudo systemctl restart seaboat-gunicorn
```

## ⚠️ 最常见的 3 个问题

### 1️⃣ 502 Bad Gateway
**原因：** Gunicorn 未运行
```bash
sudo systemctl start seaboat-gunicorn
sudo systemctl status seaboat-gunicorn
```

### 2️⃣ 数据库连接错误
**原因：** .env 文件密码错误
```bash
cat /home/wwwroot/yunpingtai.cc/.env | grep DB_PASSWORD
mysql -u seaboat_user -p -h localhost seaboat_prod  # 测试连接
```

### 3️⃣ 前端打不开或提示 404
**原因：** 前端未构建或 Nginx 配置错误
```bash
ls /home/wwwroot/yunpingtai.cc/art-design-pro/dist/index.html

# 如果不存在，重新构建：
cd /home/wwwroot/yunpingtai.cc/art-design-pro
npm run build
```

## 💡 部署小提示

1. **保存 SECRET_KEY**
   - 用于 Django session 和 token 加密
   - 泄露会导致安全风险
   - 建议保存到密码管理器

2. **数据库备份**
   ```bash
   # 每周定期备份
   mysqldump -u seaboat_user -p seaboat_prod > /home/backup/seaboat_backup_$(date +%Y%m%d).sql
   ```

3. **定期更新**
   ```bash
   # 检查更新（每月1次）
   sudo yum update -y
   
   # 更新 Python 包
   source /home/wwwroot/yunpingtai.cc/venv/bin/activate
   pip list --outdated
   ```

4. **监控磁盘**
   ```bash
   df -h  # 查看磁盘使用情况
   ```

5. **SSL 证书续期**
   - 宝塔面板已配置自动续期
   - 或手动：`sudo certbot renew`
