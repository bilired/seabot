# 调试注册 400 错误

## 🔍 可能的原因

### 1️⃣ Django 服务器没有重启
你修改了代码后，需要重启 Django 服务器。

**解决步骤**：
1. 在 Django 终端按 `CTRL+C` 停止服务器
2. 重新启动：
   ```bash
   cd /Users/hanksgao/Desktop/art-design-menu/back-end/seadrone
   python manage.py runserver 0.0.0.0:8000
   ```

### 2️⃣ 前端发送的数据格式不对
检查浏览器开发者工具 (F12) → Network 标签：
- 点击 `POST /api/register/` 请求
- 查看 "Request" 或 "Request Payload" 标签
- 应该看到：
```json
{
  "userName": "testuser",
  "password": "password123",
  "confirmPassword": "password123"
}
```

如果字段名或格式不对，可能是前端代码问题。

### 3️⃣ 查看后端错误日志
在 Django 运行的终端中，应该能看到：
```
[24/Jan/2026 18:20:53] "POST /api/register/ HTTP/1.1" 400 ...
```

上面应该有详细的错误信息。

## 🧪 快速测试

先用 curl 直接测试后端：

```bash
curl -X POST http://localhost:8000/api/register/ \
  -H "Content-Type: application/json" \
  -d '{
    "userName": "testuser123",
    "password": "password123",
    "confirmPassword": "password123"
  }' | python -m json.tool
```

**成功响应应该是**：
```json
{
  "code": 200,
  "msg": "注册成功，请登录",
  "data": {
    "userId": 1,
    "userName": "testuser123"
  }
}
```

**失败响应示例**：
```json
{
  "code": 400,
  "msg": "用户名已被注册"
}
```

## ✅ 完整检查清单

1. ☐ Django 服务器已重启
2. ☐ 用 curl 测试后端能正常响应
3. ☐ 前端浏览器已硬刷新 (Cmd+Shift+R)
4. ☐ 检查 Network 标签中的请求数据格式
5. ☐ 检查 Django 终端的错误日志

## 📝 常见错误信息

| 错误信息 | 原因 | 解决 |
|---------|------|------|
| 用户名已被注册 | 用户名重复 | 使用新的用户名 |
| 两次密码输入不一致 | 密码不一致 | 确保两次密码相同 |
| 密码长度不能少于 6 个字符 | 密码太短 | 至少 6 个字符 |
| 用户名长度必须在 3-30 个字符之间 | 用户名长度不符 | 3-30 字符 |
