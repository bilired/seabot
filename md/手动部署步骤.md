# SeaBoat 手动部署步骤（不依赖 deploy.sh）

适用场景：
- 前端在本地构建完成后上传 `dist` 到服务器
- 后端在服务器手动安装、迁移、启动
- 全程不用 `bash deploy.sh`

---

## 0. 服务器信息约定

请按你的真实环境替换：

- 项目根目录：`/www/wwwroot/yunpingtai.cc`
- 后端目录：`/www/wwwroot/yunpingtai.cc/back-end/seadrone`
- 前端目录：`/www/wwwroot/yunpingtai.cc/art-design-pro`
- 域名：`yunpingtai.cc`
- Python：`python3.11`（建议）

## 1. 本地构建前端并上传 `dist`

### 1.1 本地构建

```bash
cd /Users/hanksgao/Desktop/seabot/art-design-pro

# 二选一
pnpm install
pnpm build

# 或
npm install
npm run build
```

构建成功后应有：

```bash
ls -la dist
```

### 1.2 上传 dist 到服务器

方式 A（推荐，覆盖式同步）：

```bash
rsync -avz --delete dist/ root@8.130.132.91:/www/wwwroot/yunpingtai.cc/art-design-pro/dist/
```

方式 B（scp）：

```bash
scp -r dist root@8.130.132.91:/www/wwwroot/yunpingtai.cc/art-design-pro/
```

## 2. 服务器准备后端环境

```bash
ssh root@8.130.132.91
cd /www/wwwroot/yunpingtai.cc
```

### 2.1 检查 Python / Node / Nginx

```bash
python3.11 --version || python3 --version
node --version
nginx -v
```

### 2.2 创建虚拟环境并安装依赖

```bash
cd /www/wwwroot/yunpingtai.cc

# 首次执行
python3.11 -m venv venv

source venv/bin/activate
pip install --upgrade pip setuptools wheel

pip install Django==5.2.10
pip install djangorestframework==3.16.1
pip install djangorestframework-simplejwt==5.5.1
pip install django-cors-headers==4.9.0
pip install gunicorn==23.0.0
pip install pymysql
pip install psycopg2-binary==2.9.10
pip install alibabacloud_dysmsapi20170525
pip install alibabacloud_tea_openapi

# 让 Django 使用 PyMySQL 充当 MySQL 驱动
cat > /www/wwwroot/yunpingtai.cc/back-end/seadrone/seadrone/__init__.py << 'EOF'
import pymysql
pymysql.install_as_MySQLdb()
EOF
```

## 3. 配置 `.env`（关键）

必须在项目根目录创建：`/www/wwwroot/yunpingtai.cc/.env`

```bash
cd /www/wwwroot/yunpingtai.cc
cp .env.example .env
```

至少确认以下项已正确：

```dotenv
DB_PASSWORD=你的数据库密码
SECRET_KEY=你的Django密钥
ALIYUN_SMS_ACCESS_KEY_ID=...
ALIYUN_SMS_ACCESS_KEY_SECRET=...
ALIYUN_SMS_SIGN_NAME=...
ALIYUN_SMS_TEMPLATE_CODE=...
```

快速自检：

```bash
grep -E '^(DB_PASSWORD|SECRET_KEY|ALIYUN_SMS_ACCESS_KEY_ID|ALIYUN_SMS_ACCESS_KEY_SECRET|ALIYUN_SMS_SIGN_NAME|ALIYUN_SMS_TEMPLATE_CODE)=' .env
sed -i 's/\r$//' .env
```

## 4. 手动执行后端迁移与静态收集

```bash
cd /www/wwwroot/yunpingtai.cc
source venv/bin/activate

export DJANGO_SETTINGS_MODULE=seadrone.settings_prod
export SMS_MOCK_ENABLED=false

mkdir -p logs
touch logs/django.log logs/gunicorn_access.log logs/gunicorn_error.log
chmod 755 logs
chmod 644 logs/*.log

cd back-end/seadrone

python manage.py migrate --settings=seadrone.settings_prod
python manage.py createsuperuser --settings=seadrone.settings_prod
python manage.py collectstatic --noinput --settings=seadrone.settings_prod
```

## 5. 配置并启动 Gunicorn（systemd）

在项目根目录生成服务文件（或直接使用已有 `seaboat-gunicorn.service`）：

```bash
cd /www/wwwroot/yunpingtai.cc
sed -i "s#__PROJECT_ROOT__#/www/wwwroot/yunpingtai.cc#g" seaboat-gunicorn.service
```

安装并启动：

```bash
sudo cp /www/wwwroot/yunpingtai.cc/seaboat-gunicorn.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable seaboat-gunicorn
sudo systemctl restart seaboat-gunicorn
sudo systemctl status seaboat-gunicorn --no-pager
```

## 6. 配置 Nginx

把项目里的 `nginx.conf` 内容放到站点配置（宝塔面板里改，或命令行覆盖）。

命令行示例：

```bash
sudo cp /www/wwwroot/yunpingtai.cc/nginx.conf /usr/local/nginx/conf/vhost/yunpingtai.cc.conf
sudo nginx -t
sudo systemctl reload nginx
```

## 8. 以后更新（固定流程）

### 前端更新

1. 本地重新 `pnpm build` 或 `npm run build`
2. 重新上传 `art-design-pro/dist`
3. `sudo systemctl reload nginx`

### 后端更新

```bash
cd /www/wwwroot/yunpingtai.cc
source venv/bin/activate
cd back-end/seadrone
python manage.py migrate --settings=seadrone.settings_prod
python manage.py collectstatic --noinput --settings=seadrone.settings_prod
sudo systemctl restart seaboat-gunicorn
```

## 9. 常见报错速查

### 报错：`DB_PASSWORD 未设置`

原因：`/www/wwwroot/yunpingtai.cc/.env` 缺少该项或格式错误。

修复：

```bash
cd /www/wwwroot/yunpingtai.cc
grep -n '^DB_PASSWORD=' .env
echo 'DB_PASSWORD=你的密码' >> .env   # 如果没有
sed -i 's/\r$//' .env
```

### 报错：`502 Bad Gateway`

修复：

```bash
sudo systemctl restart seaboat-gunicorn
sudo systemctl status seaboat-gunicorn --no-pager
sudo journalctl -u seaboat-gunicorn -n 100 --no-pager
```

### 报错：`前端空白/资源404`

修复：

```bash
ls -la /www/wwwroot/yunpingtai.cc/art-design-pro/dist
sudo nginx -t && sudo systemctl reload nginx
```
