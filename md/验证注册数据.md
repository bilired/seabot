# 验证注册数据是否进入数据库

是的，**注册信息确实会进入数据库**。让我为你展示如何验证。

## ✅ 代码确认

在 `accounts/views.py` 的 `register_view` 函数中，第 68-72 行：

```python
# 8. 创建新用户
try:
    user = User.objects.create_user(
        username=username,
        password=password,
        email=email
    )
```

`User.objects.create_user()` 是 Django ORM 的标准方法，它会：
1. ✅ 自动创建用户对象
2. ✅ 自动哈希密码（使用 PBKDF2 加密）
3. ✅ 自动保存到数据库的 `auth_user` 表
4. ✅ 返回创建的用户对象

## 📊 数据库结构

Django 使用的是 SQLite 数据库（默认），用户信息存储在 `auth_user` 表：

```
db.sqlite3
├── auth_user 表
│   ├── id (主键)
│   ├── username (用户名)
│   ├── email (邮箱)
│   ├── password (加密的密码)
│   ├── first_name
│   ├── last_name
│   ├── date_joined (注册时间)
│   ├── last_login (最后登录时间)
│   └── is_staff, is_superuser, is_active...
```

## 🔒 密码加密

**重要**: 数据库中的密码是加密的，看起来像这样：

```
pbkdf2_sha256$600000$abcdef1234567890$XYZ1234567890abcdef1234567890abcdef1234567890abcdef1234567890ab
```

这是使用 Django 的 PBKDF2 算法加密的，无法直接解密。只有在登录时，Django 才会对比用户输入的密码和数据库中的哈希值。

## ✨ 完整的验证流程

```
1. 注册用户 → User.objects.create_user() → 保存到数据库

2. 用户查询 → User.objects.get() → 从数据库读取

3. 登录验证 → authenticate() → 对比密码哈希 → 成功登录
```

## 🐛 如果注册失败

如果你注册后发现用户没有出现在数据库中，检查：

1. ✅ Django 服务器是否正在运行
2. ✅ 是否看到 HTTP 201 成功响应
3. ✅ 数据库文件是否存在 (`db.sqlite3`)
4. ✅ 是否有数据库迁移错误
