# 宝塔面板部署指南 - yunpingtai.cc

**部署环境概览：**
- 域名：yunpingtai.cc
- 管理面板：宝塔面板（Baota Linux Panel）
- 系统：CentOS 8.2.2004 x86_64
- 当前Python：3.7.9
- 部署方式：前后端同一服务器

---

## ⚠️ 关键问题：Python 版本兼容性

**现状：** Django 5.2.10 需要 Python 3.10.1+，但服务器只有 Python 3.7.9

**解决方案（三选一）：**

### 方案A：在宝塔面板中安装 Python 3.11（推荐）
1. SSH 登录服务器
2. 打开宝塔面板 → 系统设置 → 软件商店
3. 搜索 "Python" → 选择 Python 3.11 安装
4. 或使用命令安装：
   ```bash
   yum install gcc openssl-devel bzip2-devel libffi-devel zlib-devel -y
   cd /tmp
   wget https://www.python.org/ftp/python/3.11.0/Python-3.11.0.tgz
   tar xzf Python-3.11.0.tgz
   cd Python-3.11.0
   ./configure --prefix=/usr/local/python311
   make && make install
   ln -s /usr/local/python311/bin/python3.11 /usr/bin/python3.11
   ```

### 方案B：使用 Docker 容器（如果宝塔支持）
将应用容器化，无需主机Python版本一致

### 方案C：降级 Django（不推荐）
如果无法升级Python，可用 Django 3.2 LTS，但会失去新特性

## 📋 部署清单

### 第一步：准备宝塔面板（UI操作）

#### 1.1 创建网站
1. 登录宝塔面板（默认 IP:8888）
2. 左边栏 → **网站** → **添加网站**
3. 配置项目：
   - **域名：** yunpingtai.cc 和 www.yunpingtai.cc
   - **建站方式：** 静态网站（前端） + 反向代理（后端）
   - **FTP：** 不需要
   - **数据库：** 选择 MySQL 8.0 / PostgreSQL（推荐用 PostgreSQL）
   - **PHP 版本：** 无（静态网站）
4. 点击 **提交**

#### 1.2 配置 SSL 证书
1. 网站列表 → 点击 yunpingtai.cc → **SSL**
2. 申请 Let's Encrypt 免费证书
3. 自动续期：勾选"自动续期"

#### 1.3 创建数据库
1. 左边栏 → **数据库**
2. **添加数据库**：
   - **数据库名：** seaboat_prod
   - **用户名：** seaboat_user
   - **密码：** [强密码，保存备用]
   - **字符集：** utf8mb4
   - **collation：** utf8mb4_unicode_ci
3. **提交**

#### 2.3 配置 Django 生产环境

#### 2.4 初始化数据库
```bash
cd /home/wwwroot/yunpingtai.cc/back-end/seadrone
source ../../venv/bin/activate

# 设置生产环境变量
export DJANGO_SETTINGS_MODULE=seadrone.settings_prod
export DB_PASSWORD='seaboat_user_password'  # 改为你的数据库密码
export SECRET_KEY='your-secure-random-key-generate-with-django-secret-key'

# 执行迁移
python manage.py migrate

# 创建超级用户
python manage.py createsuperuser

# 收集静态文件
python manage.py collectstatic --noinput

# 测试 Gunicorn
gunicorn seadrone.wsgi:application --bind 127.0.0.1:8000 --workers 4
```

#### 2.5 配置 Gunicorn systemd 服务
**创建文件：**/etc/systemd/system/seaboat-gunicorn.service**
```ini
[Unit]
Description=SeaBoat Gunicorn Application Server
After=network.target mysql.service

[Service]
Type=notify
User=www
WorkingDirectory=/home/wwwroot/yunpingtai.cc/back-end/seadrone
Environment="PATH=/home/wwwroot/yunpingtai.cc/venv/bin"
Environment="DJANGO_SETTINGS_MODULE=seadrone.settings_prod"
Environment="DB_PASSWORD=seaboat_user_password"
Environment="SECRET_KEY=your-secret-key"
ExecStart=/home/wwwroot/yunpingtai.cc/venv/bin/gunicorn \
    --workers 4 \
    --worker-class sync \
    --bind unix:/home/wwwroot/yunpingtai.cc/gunicorn.sock \
    --access-logfile /home/wwwroot/yunpingtai.cc/logs/gunicorn_access.log \
    --error-logfile /home/wwwroot/yunpingtai.cc/logs/gunicorn_error.log \
    seadrone.wsgi:application
ExecReload=/bin/kill -s HUP $MAINPID
KillMode=mixed
KillSignal=SIGTERM
Restart=on-failure
RestartSec=5s

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable seaboat-gunicorn
sudo systemctl start seaboat-gunicorn
sudo systemctl status seaboat-gunicorn
```

#### 3.3 配置 Nginx
在宝塔面板中编辑网站配置：

**访问方式：** 网站列表 → yunpingtai.cc → **配置文件**

**在 server { } 块中添加：**
```nginx
# 前端静态文件
location / {
    root /home/wwwroot/yunpingtai.cc/art-design-pro/dist;
    try_files $uri $uri/ /index.html;
    expires 30d;
    add_header Cache-Control "public, max-age=2592000";
}

# API 反向代理到 Gunicorn
location /api/ {
    proxy_pass http://unix:/home/wwwroot/yunpingtai.cc/gunicorn.sock;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_connect_timeout 60s;
    proxy_send_timeout 60s;
    proxy_read_timeout 60s;
}

# 静态文件（Django 收集的）
location /static/ {
    alias /home/wwwroot/yunpingtai.cc/static/;
    expires 30d;
}

# 媒体文件
location /media/ {
    alias /home/wwwroot/yunpingtai.cc/media/;
    expires 7d;
}
```

**保存并重启 Nginx：**
```bash
sudo systemctl reload nginx
```

## 🆘 常见问题

### Q: 上传视频时出现 413 错误
**A:** 在 Nginx 配置中添加：
```nginx
client_max_body_size 500M;
```

### Q: CSRF 错误或跨域问题
**A:** 确认 `settings_prod.py` 中的 `CSRF_TRUSTED_ORIGINS` 和 `CORS_ALLOWED_ORIGINS` 包含你的域名

### Q: 数据库连接超时
**A:** 检查 MySQL 是否运行：`sudo systemctl status mysqld`
或检查连接参数是否正确

### Q: 静态文件 404
**A:** 确保已运行 `python manage.py collectstatic` 且 STATIC_ROOT 路径正确
