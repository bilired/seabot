# 仪表板与后端集成指南

## 概述

本指南说明了如何将前端仪表板（工作台）与 Django 后端进行完整集成。

## 已完成的工作

### 后端部分

#### 1. 创建模型 (models.py)

创建了两个新模型用于存储仪表板数据：

```python
# DashboardStats: 存储用户的仪表板统计数据
- total_tasks: 总任务数
- completed_tasks: 已完成任务数
- active_projects: 活跃项目数
- total_sales: 总销售额
- new_users_count: 新用户数
- updated_at: 更新时间

# UserActivity: 存储用户活动日志
- user: 外键关联
- activity_type: 活动类型（login, task_completed 等）
- description: 活动描述
- created_at: 创建时间
```

#### 2. 创建 API 视图 (views.py)

添加了以下 API 端点：

| 端点 | 功能 |
|------|------|
| `GET /api/dashboard/stats/` | 获取仪表板统计数据 |
| `GET /api/dashboard/sales/` | 获取销售数据 |
| `GET /api/dashboard/growth/` | 获取用户增长数据 |
| `GET /api/dashboard/activity/` | 获取用户活动日志 |

#### 3. 配置路由 (urls.py)

在 `accounts/urls.py` 中配置了仪表板 API 路由。

## 常见问题解决

### 问题 1: CORS 错误

**症状**: 浏览器控制台出现 CORS 错误

**解决方案**:
1. 确保后端已启动在 `http://localhost:8000`
2. 检查 `settings.py` 中的 `CORS_ALLOWED_ORIGINS` 包含 `http://localhost:5173`
3. 重启后端服务

### 问题 2: Token 未授权错误（401）

**症状**: API 返回 401 错误

**解决方案**:
1. 确保已正确登录
2. 检查 `Authorization` 请求头格式：`Bearer <token>`
3. 检查 token 是否过期（有效期为 1 小时）
4. 如果过期，使用 `refreshToken` 获取新的 token

### 问题 3: 数据未显示

**症状**: 仪表板显示为空或默认值

**解决方案**:
1. 检查浏览器控制台是否有错误
2. 确保数据库中有测试数据
3. 检查 API 响应是否正确
4. 确保用户已登录且有访问权限

## 性能优化建议

1. **缓存**: 在前端使用 localStorage 缓存不常变化的数据
2. **分页**: 对活动日志等大量数据使用分页
3. **防抖**: 在搜索框等频繁操作的地方使用防抖
4. **虚拟滚动**: 长列表使用虚拟滚动优化性能

## 安全建议

1. **HTTPS**: 生产环境必须使用 HTTPS
2. **Token 管理**: 不要在 URL 中传递 token
3. **CSRF 保护**: 确保 Django CSRF 保护已启用
4. **输入验证**: 前后端都要验证用户输入
5. **权限检查**: 在后端验证用户权限
