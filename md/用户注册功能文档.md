# 用户注册功能文档

## ✅ 后端注册 API 已完成

### API 接口

#### 注册端点
```
POST /api/register/
Content-Type: application/json

请求体:
{
  "userName": "newuser",
  "password": "123456",
  "confirmPassword": "123456",
  "email": "user@example.com"  // 可选
}

成功响应 (201 Created):
{
  "code": 200,
  "msg": "注册成功，请登录",
  "data": {
    "userId": 2,
    "userName": "newuser",
    "email": "user@example.com"
  }
}

失败响应示例 (400 Bad Request):
{
  "code": 400,
  "msg": "用户名已被注册"
}
```

## 📝 注册功能的验证规则

1. **用户名**
   - ✅ 必填
   - ✅ 长度 3-30 个字符
   - ✅ 不能重复

2. **密码**
   - ✅ 必填
   - ✅ 长度至少 6 个字符
   - ✅ 需要与确认密码相同

3. **邮箱**
   - ⚠️ 可选
   - ✅ 如果提供则不能重复

```typescript
export function fetchRegister(data: Api.Auth.RegisterParams) {
  return request.post<Api.Auth.RegisterResponse>({
    url: '/api/register/',
    data
  })
}
```

### 2. 类型定义

**文件**: `art-design-pro/src/types/api/api.d.ts`

```typescript
namespace Auth {
  interface RegisterParams {
    userName: string
    password: string
    confirmPassword: string
    email?: string
  }

  interface RegisterResponse {
    userId: number
    userName: string
    email: string
  }
}
```

## 📱 完整的注册流程示例

### 前端调用示例

```typescript
import { fetchRegister } from '@/api/auth'

// 在注册页面或组件中
const handleRegister = async () => {
  try {
    const response = await fetchRegister({
      userName: 'newuser',
      password: '123456',
      confirmPassword: '123456',
      email: 'user@example.com'
    })

    if (response.code === 200) {
      // 注册成功
      console.log('注册成功！新用户ID:', response.data.userId)
      // 跳转到登录页
      router.push({ name: 'Login' })
    }
  } catch (error) {
    // 处理注册失败
    console.error('注册失败:', error.message)
  }
}
```

## 📊 数据库变化

注册时，Django 会在 `auth_user` 表中创建新记录：

```
ID | username  | email              | password (hashed) | ...
---|-----------|-------------------|------------------|-----
1  | Admin     | admin@example.com  | pbkdf2_sha256$... |
2  | testuser  | test@example.com   | pbkdf2_sha256$... |
3  | newuser   | user@example.com   | pbkdf2_sha256$... |
```
