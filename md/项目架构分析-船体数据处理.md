# Seabot 项目架构分析 - 船体数据处理与实时更新

## 总体发现

seabot 是一个无人船监测平台，包含水质监测、营养盐检测、船体状态追踪等功能。船体数据通过 TCP 网关接收，目前采用**轮询模式**而非实时推送。

---

## 1. 船体数据模型是否包含经纬度字段？

### ✅ **结论：包含，但不在Django模型中存储**

- **DroneDevice 模型** (accounts/models.py，L97)
  - 无 latitude/longitude 字段
  - 仅存储设备静态信息：型号、长度、重量、最高航速、功能模块等
  - 这是设备元数据模型

- **船体位置数据** 通过 **TCP 数据包** 动态传输，结构定义在 `parse_boat_payload()`：
  ```python
  # 格式: shipModel|boatTimestamp|status|latitude|longitude|speed|direction|batteryVoltage
  # 例: DL-3022|20250311120000|正常|30.5234|114.305|2.5|180|24.5
  ```

#### 2.4 数据包解析流程

**步骤1：接收原始数据 (`_handle_client_connection()`, L406)**

**步骤2：二进制数据包解析 (`parse_packets()`, L60)**

数据包格式（自定义二进制协议）：
```
+---+---+------+-------+-------+-------+----+
|$@| TYPE|MSG_ID|PAD_LEN|PAYLOAD|......|CRC |
|2B| 2B  | 4B  | 4B    |variable|    2B |
+---+---+------+-------+-------+----+----+
```

- 包头：`$@`（0x24 0x40）
- 类型：W(水质) / Y(营养盐) / B(船体) / D(深度) / 0(RTK)
- CRC 校验：MODBUS RTK公式

```python
def parse_packets(data: bytes) -> Tuple[List[dict], bytes]:
    packets: List[dict] = []
    index = 0
    
    while index + 12 <= total_length:
        if data[index:index + 2] != b'$@':  # 寻找包头
            index += 1
            continue
        
        packet_type = data[index + 2:index + 4].decode('ascii')  # 获取类型
        message_id = struct.unpack('<i', data[index + 4:index + 8])[0]
        padded_length = struct.unpack('<i', data[index + 8:index + 12])[0]
        payload_len = padded_length - 2
        
        payload_bytes = data[12:12 + payload_len]
        calculated_crc = calc_crc(0, packet_data, len(packet_data) - 2)
        
        packets.append({
            'packet_type': packet_type,
            'payload_text': payload_bytes.decode('ascii', errors='ignore'),
            'payload_hex': payload_bytes.hex(),
            'crc_valid': int(calculated_crc == current_crc),
        })
```

**步骤3：有效负载解析 (`parse_boat_payload()`, L161)**

船体数据包B解析：
```python
def parse_boat_payload(input_text: str) -> dict:
    """
    预期格式（8字段）:
    shipModel|boatTimestamp|status|latitude|longitude|speed|direction|batteryVoltage
    """
    parts = input_text.strip().split('|')
    
    return {
        'ship_model': parts[0].strip(),              # 船号: DL-3022
        'boat_timestamp': parts[1] or None,          # 设备时间: YYYYMMDDhhmmss
        'status': parts[2] or None,                  # 状态: 正常/故障
        'latitude': float(parts[3]) if parts[3] else None,    # 纬度: 30.5234°
        'longitude': float(parts[4]) if parts[4] else None,   # 经度: 114.305°
        'speed': float(parts[5]) if parts[5] else None,       # 航速: 2.5节
        'direction': float(parts[6]) if parts[6] else None,   # 航向: 180°
        'battery_voltage': float(parts[7]) if parts[7] else None,  # 电压: 24.5V
        'raw': input_text.strip(),                   # 原始文本
    }
```

**步骤4：数据持久化 (`_persist_packets()`, L437)**

```python
def _persist_packets(self, ship_port: int, packets: List[dict]) -> None:
    # 优先处理 B 类数据包（船体位置）
    for packet in packets:
        if packet['packet_type'] != 'B':
            continue
        try:
            boat_payload = parse_boat_payload(packet['payload_text'])
            # 缓存最新的船体数据包（内存）
            self.last_boat_packet_by_port[ship_port] = boat_payload
            self.reported_ship_model_by_port[ship_port] = boat_payload['ship_model']
        except Exception as exc:
            logger.warning(f'Parse boat packet failed on port {ship_port}: {exc}')
    
    # 处理其他类型数据包
    for packet in packets:
        packet_type = packet['packet_type']
        if packet_type == 'W':
            # 水质数据 -> 写入 WaterQualityData 表
            model_data = parse_water_payload(ship_model, packet['payload_text'])
            WaterQualityData.objects.create(**model_data)
        elif packet_type in ('Y', '@'):
            # 营养盐 -> 写入 NutrientData 表
            model_data = parse_nutrient_payload(ship_model, packet['payload_text'])
            NutrientData.objects.create(**model_data)
        elif packet_type == 'D':
            # 深度数据 -> 内存缓存
            depth_payload = parse_depth_payload(packet['payload_text'])
            self.last_depth_packet_by_port[ship_port] = depth_payload
        elif packet_type == '0':
            # RTK 定位 -> 内存缓存
            rtk_payload = parse_rtk_payload(packet['payload_text'])
            self.last_rtk_packet_by_port[ship_port] = rtk_payload
```

## 5. 是否有 WebSocket 或实时推送机制？

### ❌ **否。采用轮询模式**

#### 当前架构的限制：

1. **船体位置数据** 仅缓存在内存中
   ```python
   # ship_gateway.py - L298
   self.last_boat_packet_by_port: Dict[int, dict] = {}  # 仅内存存储
   ```

2. **获取方式**：HTTP 轮询
   ```
   GET /api/ship/gateway/status/
   └─> 返回网关中缓存的最新数据包
   ```

3. **更新频率**：完全取决于
   - 服务器的轮询间隔
   - 船只的数据报文发送频率（通常每秒一次）

#### 数据流现状

```
┌─────────────────┐
│  无人船设备     │
│  (DL-3022)      │
└────────┬────────┘
         │ TCP 连接 (端口 9001)
         ▼

│ ShipGatewayService              │
│ ┌─────────────────────────────┐ │
│ │ last_boat_packet_by_port    │ │  <- 内存缓存
│ │ {9001: {lat, lon, ...}}     │ │
```

## 当前架构的优缺点

### ✅ 优点
- 轻量级，无需复杂的消息队列
- TCP 长连接保证数据不丢失
- 支持多船多端口并行管理
- 协议简单高效（二进制 + CRC 校验）

### ❌ 缺点
1. **无实时推送** - 前端必须轮询，延迟高
2. **内存存储** - 只保留最新一条数据，历史轨迹丢失
3. **无Web持久化** - 船体位置不写入数据库
4. **单机架构** - 无法横向扩展
5. **无认证** - 所有API都允许匿名访问

---

## 建议的改进方向

### 短期（快速见效）
1. 实现 WebSocket 实时推送 (Django Channels)
2. 船体位置持久化到数据库（BoatTrajectory 表）
3. 前端添加地图组件显示实时位置

### 中期（稳定运维）
1. 使用 Redis 替代内存存储
2. 实现轨迹回放功能
3. 添加异常告警（离线、电压低等）

### 长期（高可用）
1. 微服务架构（网关独立部署）
2. 分布式消息队列（RabbitMQ/Kafka）
3. 时序数据库（InfluxDB）存储位置时间序列
