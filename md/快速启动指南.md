# 🚀 快速启动指南

## 一、项目结构

```
seabot/
├── art-design-pro/           # 前端项目 (Vue 3)
├── back-end/                 # 后端项目 (Django)
│   └── seadrone/            # Django 应用
├── .venv/                    # Python 虚拟环境
└── startup.sh               # 启动脚本
```

## 二、快速启动（推荐）

### 使用启动脚本（最简单）

```bash

# 运行启动脚本
./startup.sh
```

然后按照菜单选择你要执行的操作：
- **选项 1**: 启动后端服务 (Django)
- **选项 2**: 启动前端服务 (Vue 3)
- **选项 5**: 查看数据库内容

## 三、手动启动

### 方案 A：仅启动后端

```bash
# 进入后端目录
cd /Users/hanksgao/Desktop/seabot/back-end/seadrone

# 激活虚拟环境
source ../../.venv/bin/activate

# 启动 Django 开发服务器
python3 manage.py runserver 0.0.0.0:8000
```

✅ 后端已启动: http://localhost:8000

---

### 方案 B：仅启动前端

```bash
# 进入前端目录
cd /Users/hanksgao/Desktop/seabot/art-design-pro

# 安装依赖（如果还未安装）
pnpm install

# 启动开发服务器
pnpm dev
```

✅ 前端已启动: http://localhost:5173

---

### 方案 C：同时启动前后端（推荐）

**在终端 1 中启动后端：**
```bash
cd /Users/hanksgao/Desktop/seabot/back-end/seadrone
source ../../.venv/bin/activate
python3 manage.py runserver 0.0.0.0:8000
```

**在终端 2 中启动前端：**
```bash
cd /Users/hanksgao/Desktop/seabot/art-design-pro
pnpm dev
```

## 八、常见问题

### Q1: 端口 8000 已被占用？

```bash
# 查看占用 8000 端口的进程
lsof -i :8000

# 杀死占用该端口的进程
kill -9 <PID>
```

### Q2: 前端依赖未安装？

```bash
cd /Users/hanksgao/Desktop/seabot/art-design-pro
pnpm install
```

### Q3: 虚拟环境激活失败？

```bash
# 检查虚拟环境
cd /Users/hanksgao/Desktop/seabot
source .venv/bin/activate

# 如果失败，重新创建虚拟环境
python3 -m venv .venv
source .venv/bin/activate
pip install -r back-end/requirements.txt
```

### Q4: 数据库错误？

```bash
# 进入后端目录
cd /Users/hanksgao/Desktop/seabot/back-end/seadrone

# 运行迁移
python3 manage.py migrate

# 创建超级用户
python3 manage.py createsuperuser
```
