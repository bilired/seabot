# 修复：添加用户信息接口

## ✅ 已修复的问题

登录成功后，前端尝试获取用户信息时返回 404，原因是后端还没有相应的接口。

## 🔧 修复内容

### 1. 后端新增 `user_info_view()` 接口
**文件**: `back-end/seadrone/accounts/views.py`

新增了获取用户信息的视图：
```python
@api_view(['GET'])
@permission_classes([IsAuthenticated])  # 需要身份验证
def user_info_view(request):
    """获取当前登录用户的信息"""
    user = request.user
    
    return Response({
        "code": 200,
        "msg": "获取成功",
        "data": {
            "userId": user.id,
            "userName": user.username,
            "email": user.email,
            "avatar": "",
            "roles": [],
            "buttons": []
        }
    })
```

### 2. 更新路由配置
**文件**: `back-end/seadrone/accounts/urls.py`

添加了新路由：
```python
urlpatterns = [
    path('login/', views.login_view, name='login'),
    path('user/info/', views.user_info_view, name='user_info'),  # ✨ 新增
]
```

### 3. 前端 URL 统一
**文件**: `art-design-pro/src/api/auth.ts`

确保末尾都有斜杠：
```typescript
export function fetchGetUserInfo() {
  return request.get<Api.Auth.UserInfo>({
    url: '/api/user/info/'  // ✅ 添加末尾斜杠
  })
}
```

## 📋 API 总结

现在的完整 API 流程：

| 步骤 | 请求方式 | URL | 认证 | 说明 |
|------|---------|-----|------|------|
| 1 | POST | `/api/login/` | ❌ | 登录，返回 token |
| 2 | GET | `/api/user/info/` | ✅ | 获取用户信息 |

## 🐛 如果还有问题

检查浏览器开发者工具的 Network 标签，应该看到两个请求：

```
1. POST /api/login/ → 200 (成功，返回 token)
2. GET /api/user/info/ → 200 (成功，返回用户信息)
```

如果第二个请求仍然返回 404，可能原因：
- ❌ Django 服务器没有重启（新路由未加载）
- ❌ 浏览器缓存没有清除
- ❌ 路由配置有误

例如：
```python
from django.contrib.auth.models import Group

def user_info_view(request):
    user = request.user
    groups = user.groups.all().values_list('name', flat=True)
    
    return Response({
        "code": 200,
        "data": {
            "userId": user.id,
            "userName": user.username,
            "roles": list(groups),
            # ... 其他信息
        }
    })
```
