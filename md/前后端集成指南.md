## 📝 API 对接详情

### 登录请求
- **URL**: `/api/auth/login` (前端) → `/api/login/` (后端)
- **方法**: POST
- **Content-Type**: application/json
- **请求体**:
```json
{
  "userName": "Admin",
  "password": "123456"
}
```

### 登录响应
```json
{
  "code": 200,
  "msg": "登录成功",
  "data": {
    "token": "eyJ0eXAiOiJKV1QiLCJhbGc...",
    "refreshToken": "eyJ0eXAiOiJKV1QiLCJhbGc..."
  }
}
```

### 前端处理流程
1. `src/views/auth/login/index.vue` - 登录页面
2. 调用 `fetchLogin()` (在 `src/api/auth.ts`)
3. HTTP 拦截器 (在 `src/utils/http/index.ts`) 自动:
   - 添加 Authorization 请求头
   - 处理响应错误
   - 401 自动登出
4. 存储 token 到 `useUserStore()`
5. 跳转到首页

## 🔧 常见问题

### 问题 1：CORS 错误
**症状**：浏览器控制台显示 `No 'Access-Control-Allow-Origin' header`

**解决**：
1. 确保安装了 `django-cors-headers`
2. 确认 `CORS_ALLOWED_ORIGINS` 中包含你的前端地址
3. 重启 Django 服务器

### 问题 2：404 Not Found
**症状**：请求返回 404

**解决**：
1. 确认 Django urls.py 中包含了 `path('api/', include('accounts.urls'))`
2. 确认后端接口是 `/api/login/` (注意末尾的斜杠)

### 问题 3：登录失败 "用户名或密码错误"
**症状**：输入正确的用户名密码仍然失败

**解决**：
1. 创建测试用户：
```bash
python manage.py createsuperuser
# 或
python manage.py shell
>>> from django.contrib.auth.models import User
>>> User.objects.create_user('Admin', password='123456')
```

2. 确认数据库中有用户数据

### 问题 4：Token 格式错误
**症状**：登录成功但后续请求失败

**解决**：
1. 检查 `src/utils/http/index.ts` 中的 Authorization 请求头格式
2. 确保是 `Authorization: {token}` 或 `Authorization: Bearer {token}`
